# 企业级皮肤大数据管理与智能分析平台 PRD v1

## 1. 产品定位

构建一个面向企业客户的多租户 SaaS 平台，支持皮肤相关数据的安全接入、标准化处理、统计分析、科研级可视化输出和结果追溯。平台需要同时满足以下目标：

- 企业级多租户隔离与权限治理
- 面向业务与科研双场景的数据分析能力
- 面向中文用户的低学习成本工作流体验
- 面向科研输出的高质量图表与结果下载能力

---

## 2. 核心业务场景

### 2.1 典型用户角色

- 平台超级管理员：管理租户、套餐、系统参数、审计策略
- 租户管理员：管理本企业用户、项目、数据权限、分析权限
- 数据上传员：上传数据、字段映射、预处理、修正数据问题
- 分析师/研究员：创建分组、运行统计分析、配置图表、导出结果
- 业务查看者：查看报表、下载结果、追溯分析过程
- 审计/合规人员：查看审计日志、导出操作记录

### 2.2 核心流程

1. 租户开通与组织初始化
2. 创建项目/课题/数据空间
3. 上传 Excel/CSV/文本模板文件
4. 字段识别、编码识别、校验与预处理
5. 通过条件筛选与分组合并构建分析样本
6. 选择分析方法并自动执行前提检验
7. 查看统计结果、图表、显著性与效应量
8. 下载结果文件、图像文件、过程文件和方法说明
9. 留存全过程审计记录

## 3. 功能架构

### 3.1 租户与组织管理

- 支持多租户组织开通，每个客户组织拥有独立数据空间
- 支持组织、部门、项目三级管理
- 支持 RBAC 权限模型
- 支持项目级数据访问控制
- 支持单点登录扩展能力（企业微信、钉钉、Azure AD、OIDC/SAML 预留）
- 支持审计日志、登录日志、数据下载日志

### 3.2 数据上传与数据接入

- 支持 Excel、CSV 上传
- 支持固定格式文本文件上传，用于特定比较分析输入
- 兼容中文字段名
- 兼容 GBK、UTF-8 等常见编码自动识别
- 支持模板校验、字段映射、数据类型推断
- 支持批量上传任务和失败行回显
- 支持样本主键唯一性校验、重复记录检测

### 3.3 数据预处理

- 缺失值识别与处理建议
- 异常值标注与异常规则说明
- 数值型/分类型变量识别
- 单位规范化与范围校验
- 字段标准化字典管理
- 预处理过程可追溯，可回放，可下载

### 3.4 分组与筛选引擎

- 支持单条件筛选
- 支持多条件组合筛选
- 支持客观指标四分位分组
- 支持问卷选项与选项组合筛选
- 支持条件逻辑组合：且/或/非
- 支持分组规则保存与复用
- 支持条件合并计算，例如“中度+重度”对比“轻度+无”
- 支持筛选结果样本量即时预览

### 3.5 统计分析引擎

#### 描述性统计

- 均值、中位数、标准差、四分位数、极值、偏度、峰度
- 按组汇总
- 自动生成基础图表

#### 差异性分析

- t 检验
- 单因素 ANOVA
- Kruskal-Wallis
- 自动执行 Shapiro-Wilk 正态性检验
- 自动执行方差齐性检验
- 根据检验结果推荐参数法或非参数法
- 自动输出效应量
- 自动输出事后多重比较结果

#### 相关性分析

- Pearson 相关
- Spearman 相关
- 相关矩阵显著性标记
- 热图展示与排序

#### 回归分析

- 一元线性回归
- 多元线性回归
- VIF 多重共线性诊断
- 标准化系数
- 残差分析
- 逐步回归变量筛选

### 3.6 图表与可视化

- 柱形图（误差棒）
- 箱型图（抖动散点）
- 热图（聚类排序）
- 散点图（回归线）
- 折线图
- 聚类树状图
- 饼图
- 支持图标题、坐标标题、字体、字号、标签、配色、范围等配置
- 支持在线预览
- 支持 PNG 300 DPI 导出
- 支持 SVG 矢量导出

### 3.7 结果追溯与下载

- 显示统计方法说明
- 输出分析结果文件
- 输出过程文件
- 输出筛选条件与参数配置快照
- 支持报告下载与版本记录

## 4. 非功能与企业级要求

### 4.1 多租户隔离原则

建议采用“控制面共享、数据面隔离”的架构，而不是只靠 `tenant_id` 过滤：

- 租户级独立数据库或独立 schema
- 租户级对象存储目录或独立桶
- 租户级加密密钥
- 租户级缓存命名空间
- 服务层强制租户上下文注入
- 全链路审计与跨租户访问拦截

对于高敏客户，建议支持“独享实例/独立部署”模式。

### 4.2 安全与合规

- 全链路 HTTPS
- 静态数据加密
- 敏感文件上传病毒扫描
- 数据脱敏展示
- 操作审计与下载审计
- 权限最小化
- 支持数据保留周期与删除策略
- 支持 IP 白名单、MFA、登录风控扩展

### 4.3 性能与可靠性

- 大文件分片上传
- 异步分析任务队列
- 长任务状态跟踪
- 图表与结果缓存
- 失败重试与任务幂等
- 关键操作告警

## 5. 页面与 UX 结构

### 5.1 信息架构

- 登录页
- 租户工作台
- 项目列表页
- 数据集列表页
- 数据上传向导页
- 数据预处理页
- 分组筛选构建页
- 分析中心页
- 图表工作台页
- 结果与下载页
- 审计日志页
- 租户管理后台

### 5.2 推荐交互方式

- 上传采用向导式流程：上传 -> 校验 -> 字段映射 -> 预处理 -> 入库
- 分析采用工作台流程：选数据 -> 选变量 -> 选分组 -> 选方法 -> 看结果
- 每一步都展示“当前样本量”和“当前规则”
- 所有分析任务提供“方法说明”和“适用条件提示”
- 图表编辑区和结果区左右分栏，降低跳转成本
- 支持保存分析模板，减少重复配置

### 5.3 UI 风格建议

建议采用“医疗科研企业平台”风格，而不是泛后台风格：

- 主色建议：深青蓝、灰白、低饱和肤色辅助色
- 强调专业、可信、克制，不做娱乐化视觉
- 图表区域优先保证可读性和导出一致性
- 中文默认字体和英文字体分开配置
- 关键结果卡片突出“显著性、效应量、样本量、方法”

## 6. 数据模型建议

核心实体建议如下：

- Tenant：租户
- OrganizationUser：组织用户
- RolePermission：角色权限
- Project：项目/课题
- Dataset：数据集
- DatasetVersion：数据版本
- VariableMeta：字段元数据
- Subject：研究对象/样本
- VisitRecord：时间点/随访记录
- QuestionnaireResult：问卷结果
- MetricResult：客观指标结果
- CohortRule：分组规则
- AnalysisJob：分析任务
- AnalysisResult：分析结果
- ChartConfig：图表配置
- ExportFile：导出文件
- AuditLog：审计日志

## 7. 技术架构建议

### 7.1 推荐技术选型

前端：

- Next.js + React
- TypeScript
- Ant Design / 自定义企业设计系统
- ECharts 用于交互式图表

后端：

- Python FastAPI 作为分析服务与 API 网关核心
- Celery/RQ + Redis 处理异步分析任务
- PostgreSQL 存储业务数据
- 对象存储用于原始文件、导出文件、过程文件

分析引擎：

- pandas
- scipy
- statsmodels
- scikit-learn
- seaborn / matplotlib / plotnine 用于高质量科研导出图

### 7.2 一个关键实现原则

在线交互图和科研导出图建议分层：

- 前端图表库负责在线交互和快速预览
- 后端 Python 绘图负责 300 DPI PNG、SVG、论文级样式统一导出

这样可以避免浏览器图表导出质量不稳定的问题。

## 9. 我建议优先补充的业务规则

当前功能已经比较完整，但正式进入设计前，还需要明确以下规则：

- 一个租户下是否允许多个项目相互隔离
- 数据对象是“人”为主，还是“人-时间点”为主
- 是否包含图片/仪器检测文件这类非结构化数据
- 上传模板是否按客户行业/项目类型区分
- 逐步回归的算法规则是否允许用户配置
- 科研导出图是否需要固定期刊模板
- 是否需要审批流和双人复核
