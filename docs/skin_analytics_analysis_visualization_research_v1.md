## 2. 当前应覆盖的分析方法族

### 2.1 样本画像与分类分布

- 频数分析
- 构成比分析
- 样本画像表

专业要求：

- 输出 `N / 百分比 / 有效百分比 / 累计百分比`
- 支持按品牌、城市、年龄组、项目组等分层
- 缺失值必须单独列示

### 2.2 描述统计

- 均值
- 中位数
- 标准差
- 四分位数
- 极值
- 偏度
- 峰度

专业要求：

- 连续变量支持按组汇总
- 默认输出样本量与缺失率

### 2.3 差异分析

- 两组比较：
  `t` 检验 / `Welch t` / `Mann-Whitney`
- 多组比较：
  `ANOVA` / `Welch ANOVA` / `Kruskal-Wallis`

专业要求：

- 必须先做前提检验
- 必须输出效应量
- 必须支持事后多重比较
- 必须保留系统推荐方法与用户覆盖留痕

### 2.4 相关分析

- Pearson
- Spearman
- 分层相关分析

专业要求：

- 支持完整矩阵与交叉矩阵
- 支持显著性标注
- 支持阈值过滤摘要输出

### 2.5 回归与多因素分析

- 一元线性回归
- 多元线性回归
- 逐步回归
- GLM

专业要求：

- 支持 VIF
- 支持残差分析
- 支持 Q-Q / P-P 图
- 支持系数森林图
- GLM 需支持 `Gamma + log link`

### 2.6 降维与分群

- 因子分析
- 聚类分析

专业要求：

- 因子分析需支持：
  `KMO / Bartlett / 载荷矩阵 / 解释率 / 因子得分`
- 聚类分析需支持：
  聚类数量、类别分布、亚型画像、Z-score 特征

### 2.7 问卷专项分析

- 问卷总分/维度分
- 列联分析
- 卡方检验
- 标准化残差分析

专业要求：

- 问卷规则必须先完成计分
- 列联分析输出热图、堆叠条形图、结构图

## 3. 当前应覆盖的可视化方法族

### 3.1 差异分析主图

- 柱形图 + 误差棒
- 箱线图 + 抖动散点

专业要求：

- 误差棒可切换 `SE / SD / CI`
- 显著性横杠与显著性标签统一

### 3.2 相关与矩阵图

- 热力图
- 聚类热图

专业要求：

- 支持连续色和发散色
- 支持聚类排序
- 支持显著性标注

### 3.3 回归与诊断图

- 散点图 + 回归线
- 残差图
- Q-Q 图
- P-P 图
- 系数森林图

专业要求：

- 回归线与置信区间统一风格
- 森林图必须同时展示点估计与 95% CI

### 3.4 问卷与分类图

- 条形图
- 堆叠条形图
- 饼图
- 标准化残差热力图

### 3.5 因子与聚类图

- 因子载荷热图
- 方差解释率图
- 聚类结果图

## 4. 专业实践建议

### 4.1 方法选择

- 优先系统推荐，不让普通用户先学统计
- 普通用户只看“快速分析”
- 分析师进入“高级分析”

### 4.2 结果表达

- 每个结果页至少有：
  结果摘要
  主统计表
  主图
  方法说明
  执行快照摘要

### 4.3 图表设计

- 在线预览与科研导出必须分层
- 在线图用于探索
- 后端图用于稳定导出

### 4.4 统计可复现性

- 所有方法都要绑定 `ExecutionManifest`
- 所有方法都要有黄金数据集和容差

### 4.5 统计报告规范

每类方法结果至少应输出：

- 样本量
- 缺失值说明
- 方法名称
- 前提检验结果
- 主统计量
- `p` 值
- 效应量
- 图表
- 简版文字解释

### 4.6 科研图输出规范

- 默认支持 `PNG 300 DPI` 与 `SVG`
- 图中应支持样本量标签
- 差异分析图支持显著性标注
- 相关/残差热图应避免只靠颜色表达方向
- 配色需兼顾色盲友好和打印友好

## 5. 实现建议

### 5.1 推荐实现栈

- 统计：`scipy + statsmodels + scikit-learn`
- 图表导出：`matplotlib + seaborn`
- 在线预览：`ECharts`

### 5.2 statsmodels / seaborn 实践要点

- `statsmodels` 适合：
  回归、GLM、Q-Q 图、VIF、部分统计检验
- `seaborn` 适合：
  箱线图、热力图、分类比较图、风格统一

### 5.3 图表工作台建议

- 差异分析默认主图：误差线图
- 相关分析默认主图：热力图
- 回归诊断默认高级图：Q-Q / P-P
- 因子分析默认主图：载荷热图
- 问卷列联默认主图：标准化残差热图

### 5.4 方法输出与实现边界

建议按两层实现：

- 统计层：产出结构化结果 JSON
- 表达层：将结构化结果渲染为表、图、解释文本

这样可以避免每个方法把统计计算和页面展示耦合在一起。
