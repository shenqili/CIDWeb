# 企业级皮肤大数据管理与智能分析平台 PRD v3 复审报告

## 1. 复审结论

`PRD v3` 相比 `v2` 已经补上了以下关键缺口：

- 品牌配置进入主链路
- 多品牌项目与跨品牌混合分析规则
- 主键映射与多源整合规则
- 派生变量回写与新版本生成规则
- 数据版本状态
- 高敏导出审批角色
- 频数分析与样本画像
- 分析型复合组件和工作台版式规范
- MVP 分阶段拆分

当前版本已经可以作为 `v3` 进入下一轮原型和 ERD 设计，但仍有少量 `P1` 级问题建议在 `v3.1` 中补齐。

---

## 2. Remaining Findings

### 2.1 功能完整性

1. `P1` 多品牌项目下的“品牌切换范围”已定义，但还没有明确“品牌切换后哪些已保存对象会自动跟随切换，哪些不会”。  
对应：`4.3`、`5.1.1`、`6.5`  
建议：明确以下对象是否品牌绑定：
- 分组规则
- 分析模板
- 图表模板
- 派生变量集

2. `P1` 问卷规则支持项目级复制改写，但尚未明确“项目级问卷规则变更后，对历史结果是否重算”。  
对应：`6.2.3`、`6.2.4`  
建议：写死规则：
- 历史分析不自动重算
- 仅新分析任务使用新问卷规则版本

### 2.2 可靠性

1. `P1` `ExecutionManifest` 已提出，但字段定义还不够细。  
对应：`3`、`5.5`、`9.2`、`12.2`  
建议最少补充：
- 统计库版本
- 模板版本
- 图表模板版本
- 批次号
- 随机种子
- 时区

2. `P1` 任务状态机已写入，但缺少“取消后如何处理中间产物”的清理规则。  
对应：`7.8.1`、`11.1`、`12.2`  
建议补：
- cancelled 任务的中间文件清理策略
- failed 任务的保留时长

### 2.3 可用性 / 用户链路

1. `P1` 首次使用向导方向已经隐含成立，但未单列页面或链路。  
对应：`5.1`、`5.1.1`、`7.2`  
建议增加：
- 首次使用向导页
- 按角色的默认入口

2. `P1` 普通用户只允许单品牌分析，但分析工作台还没有“当前品牌锁定提示”设计。  
对应：`4.3`、`7.7`  
建议补：
- 工作台顶部显示当前品牌标签
- 切换品牌时提示已选变量和规则将被重置或校验

### 2.4 UI

1. `P1` 分析型复合组件已补充，但每个组件还没有“状态示例”。  
对应：`8.18`  
建议在下一版补：
- 变量投放器空态
- 规则画布冲突态
- 结果摘要卡加载态
- 导出包面板审批态

2. `P1` 图表工作台已定义三栏结构，但未指定“主图与备选图切换器”的具体位置。  
对应：`10.10`、`16.1`  
建议补：
- 主图切换器固定在中央预览区顶部
- 右侧属性面板只作用于当前主图

### 2.5 QA / 验收

1. `P1` 验收执行规则已补，但还没有形成场景化测试矩阵。  
对应：`13.1`、`13`  
建议下一步产出独立文档：
- 上传测试矩阵
- 主键映射测试矩阵
- 派生变量回写测试矩阵
- 高敏导出审批测试矩阵

2. `P1` 统计方法验收还缺“黄金样本数据集”定义。  
对应：`9`、`13.1`  
建议单独建立：
- 差异分析黄金样本
- 相关分析黄金样本
- 回归分析黄金样本
- 问卷列联黄金样本
