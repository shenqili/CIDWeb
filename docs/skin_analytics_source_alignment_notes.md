# 皮肤大数据平台数据源对齐说明

## 1. 输入文件

- `vendor_metrics.xlsx`：各品牌/各家可用指标配置
- `alldata_260204.xlsx`：统一大表原始数据
- `stat_methods.docx`：当前拟实现/参考的统计分析方法与报告表达方式

---

## 2. 从数据源得到的关键结论

### 2.1 平台必须支持“按品牌/项目配置指标可用性”

`vendor_metrics.xlsx` 证明不同品牌、不同项目的可用指标集合并不一致，不能假设所有客户共用同一套指标。

已识别品牌与范围：

- BDF：面部
- 娇韵诗：面部
- 丸美：面部
- 溪木源：面部
- 雅诗兰黛：面部
- 云南白药：头部
- 拉芳：头部
- 汉高：头部
- 环亚：面部 + 头部

按“各家指标不包含问卷”统计的非问卷指标数量：

- BDF：404
- 娇韵诗：225
- 丸美：475
- 溪木源：23
- 雅诗兰黛：23
- 云南白药：53
- 拉芳：51
- 汉高：51
- 环亚：501

这意味着系统必须存在：

- 品牌/客户配置中心
- 指标可用性矩阵
- 指标别名与映射规则
- 身体部位/测试区域配置
- 是否包含问卷的开关

### 2.2 平台数据模型不能只建“字段表”，必须建“统一指标目录”

`alldata_260204.xlsx` 是一个 7320 行、1401 列的统一宽表。它不是简单 Excel 上传页面可以解决的场景，而是典型的“主数据目录 + 动态指标治理”问题。

当前可识别的数据层次包括：

- 基础人口学：区域、城市、编号、性别、年龄、表观年龄、BMI 等
- 面部微生态：`chao1/shannon/simpson`、菌属丰度等
- 头部微生态：头部真菌、头部细菌、头皮菌属等
- 面部仪器指标：`MX18`、`Moor`、`Sebumeter`、`pH905`、`CM825`、`vapo`、`MPA580`、`CM26DG`
- 医生评估：泪沟等级、眼袋等级、眼角下垂等级等
- 头皮仪器指标：`deramlab`、`IDM800`、`TMnano`、`MB560`、`SPOCS`
- 成像/图像量化指标：`Vplus`、`Quanying`
- 大量问卷题项与业务字段

因此建议平台建立：

- `MetricCatalog`：统一指标目录
- `InstrumentFamily`：仪器/算法家族
- `BodySite`：部位/测量区域
- `MetricAvailability`：品牌-指标可用矩阵
- `MetricAlias`：原始字段名与标准字段名映射
- `QuestionnaireTemplate/Question/Option`：问卷模板、题目、选项

### 2.3 指标配置不止到“品牌”，还要细到“部位/测量点”

`vendor_metrics.xlsx` 中的 `cm26dg` sheet 表明，同一仪器在不同品牌可支持的测量点也不完全相同。

示例：

- BDF：额头、眼角、面颊
- 娇韵诗：额头、眼角、面颊
- 环亚：额头、眼角、面颊
- 丸美：仅眼角

因此配置层至少要支持：

- 品牌
- 仪器
- 测量点/部位
- 指标

## 3. 统计方法范围需要扩展

`stat_methods.docx` 说明当前真实业务范围已经超出原先 PRD 中的“描述统计 + 差异分析 + 相关 + 线性回归”。

已确认存在的方法模块：

### 3.1 差异性分析

- 按年龄分组的描述统计
- 按年龄分组的差异性分析
- 按城市分层的描述统计
- 按城市分层的差异性分析
- 箱线图输出

### 3.2 相关性分析

- 面部相关性分析
- 相关性热力图
- 分年龄组、分城市的相关性输出
- 结果摘要中存在阈值过滤逻辑，例如仅展示绝对值 > 0.2 的相关关系

### 3.3 多因素分析

文档中真实使用的不是普通线性回归，而是更接近：

- 广义线性模型 `GLM`
- Gamma 分布
- log link

输出包括：

- 模型拟合优度
- RMSE
- 显著变量数
- 系数解释表
- 回归系数森林图
- 城市、皮肤类型、日晒时长、年龄交互项等解释

### 3.4 因子与聚类分析

文档已包含：

- 因子分析
- KMO 检验
- Bartlett 球形检验
- 因子载荷矩阵热图
- 方差解释率图
- 基于因子得分的人群聚类
- 聚类结果可视化
- 聚类亚型命名与自动解释
- Z-score 高低特征输出

### 3.5 消费者问卷统计相关性分析

文档已包含：

- 问卷题项与城市等分类变量的列联分析
- 卡方检验
- 标准化残差分析
- 标准化残差热力图
- 选项结构饼图
- 堆叠条形图

## 5. 推荐的系统实现方式

### 5.1 数据接入

不要将 Excel 表头直接等价为系统字段。建议采用四层模型：

1. 原始字段名
2. 标准字段名
3. 指标目录实体
4. 品牌可用性配置

### 5.2 分析工作台

分析工作台应先经过“变量可用性裁剪”，再展示给用户：

- 先选租户/项目/品牌/数据版本
- 再根据品牌配置过滤可分析指标
- 再根据变量类型和方法规则推荐分析方法

### 5.3 结果解释

现有 Word 文档说明业务并不满足于“输出表格”，而是要求：

- 自动生成结论摘要
- 自动生成组间差异解释
- 自动生成多因素模型解释
- 自动生成聚类亚型命名与画像

因此平台应预留“结构化结果 -> 叙述模板”的能力。
