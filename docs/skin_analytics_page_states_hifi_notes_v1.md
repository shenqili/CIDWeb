## 2. 全局状态图

### 2.1 项目上下文状态

状态：

- `no_project`
- `project_bound_no_brand`
- `project_bound_brand_ready`
- `project_bound_brand_ready_no_dataset`
- `project_ready_for_analysis`

切换条件：

- 创建项目
- 绑定品牌
- 绑定 `VersionBundle`
- 发布数据版本

### 2.2 品牌上下文状态

状态：

- `brand_unselected`
- `brand_selected`
- `brand_switch_pending_confirm`
- `brand_switch_revalidated`
- `brand_switch_conflict_cleared`

关键规则：

- 切换品牌先确认
- 再做兼容性校验
- 不兼容对象清空

### 2.3 分析任务状态

状态：

- `draft`
- `queued`
- `running`
- `partial_success`
- `succeeded`
- `failed`
- `cancelled`
- `expired`

### 2.4 高敏导出审批状态

状态：

- `pending_approval`
- `approved`
- `rejected`
- `expired`

### 2.5 问卷答卷状态

状态：

- `draft`
- `submitted`
- `invalid`
- `pending_match`
- `scored`

### 2.6 关键页面状态迁移

#### 工作台迁移

- `no_project -> project_bound_no_brand`
  触发条件：创建项目
- `project_bound_no_brand -> project_bound_brand_ready`
  触发条件：绑定品牌和 `VersionBundle`
- `project_bound_brand_ready -> project_bound_brand_ready_no_dataset`
  触发条件：项目品牌已就绪但无已发布数据版本
- `project_bound_brand_ready_no_dataset -> project_ready_for_analysis`
  触发条件：存在可用数据版本

#### 品牌切换迁移

- `brand_selected -> brand_switch_pending_confirm`
  触发条件：用户点击切换品牌
- `brand_switch_pending_confirm -> brand_switch_revalidated`
  触发条件：用户确认切换并完成兼容性校验
- `brand_switch_revalidated -> brand_switch_conflict_cleared`
  触发条件：清空不兼容对象并完成上下文刷新

#### 高敏导出审批迁移

- `pending_approval -> approved`
  触发条件：租户管理员审批通过
- `pending_approval -> rejected`
  触发条件：租户管理员拒绝
- `approved -> expired`
  触发条件：下载链接超时失效

#### 问卷答卷迁移

- `draft -> submitted`
  触发条件：答卷正式提交
- `submitted -> scored`
  触发条件：计分完成
- `submitted -> pending_match`
  触发条件：无法映射 `subject_id`
- `submitted -> invalid`
  触发条件：逻辑校验失败且被判定不可用

## 3. 关键页面状态说明

### 3.1 工作台

#### 页面态

- 空状态：
  无项目
- 引导状态：
  有项目但未绑定品牌
- 就绪状态：
  有项目、有品牌、有数据
- 风险状态：
  有待审批、高风险下载、失败任务

#### 高保真建议

- 第一视觉焦点放在当前项目卡和待办卡
- 高风险提醒使用暖色但不压过主内容
- 最近任务和最近结果卡片应可点击进入详情

### 3.2 上传向导

#### 页面态

- 初始态
- 上传中
- 解析中
- 校验失败
- 主键映射冲突
- 可发布

#### 高保真建议

- 步骤条固定在顶部
- 主内容区保持单步专注
- 错误区用明确红色边框和可下载错误清单

#### 关键组件状态

- 文件上传卡：
  `idle / uploading / uploaded / parse_failed`
- 字段映射表：
  `auto_mapped / manually_adjusted / conflict`
- 主键映射卡：
  `auto_matched / manual_required / resolved`

### 3.3 快速分析工作台

#### 页面态

- 初始选择态
- 方法推荐态
- 运行中
- 结果摘要态
- 品牌切换重校验态

#### 高保真建议

- 系统推荐卡是视觉中心
- 摘要结果卡在右侧始终可见
- 高级设置入口弱化但明确

#### 与高级分析的差异

- 快速分析默认隐藏高级参数
- 快速分析只允许少量变量输入
- 快速分析运行后优先展示主图、主表和简版解释
- 快速分析不直接暴露复杂诊断标签页

### 3.3.1 全量数据明细浏览页（Data Explorer）

#### 页面态

- 初始空查询态
- 已选择品牌与版本态
- 条件筛选配置态
- 查询执行中
- 明细结果态
- 图表预览展开态
- 品牌切换重校验态
- 无结果态
- 越权品牌拦截态

#### 高保真建议

- 顶部固定项目、品牌范围、数据版本上下文带
- 左侧筛选区采用高密模式，中部表格区优先保证阅读与对齐
- 明细表上方先显示样本统计摘要，再显示数据表
- 右侧图表预览默认展示单字段分布，切换图表时保留最近一次字段选择
- 无结果与越权场景必须显示明确空状态与回退动作，不能只展示空表格

#### 角色可见性建议

- 超级管理员与租户管理员支持多品牌联合浏览
- 品牌管理员与分析师默认锁定在授权品牌范围
- 当前页面不直接给出高敏明细导出入口，导出动作应跳转审批链路

#### 页面间跳转

- 来源可为：
  数据集列表、数据版本详情、项目工作台
- 去向可为：
  快速分析工作台、高级分析工作台、结果详情页、下载中心

### 3.4 高级分析工作台

#### 页面态

- 空配置态
- 配置完成态
- 运行中
- 结果预览态
- 诊断展开态
- 参数冲突态

#### 高保真建议

- 左侧字段树采用高密模式
- 中部参数区按方法分组折叠
- 右侧优先显示摘要，再显示图表和诊断

#### 与快速分析的差异

- 高级分析暴露完整变量投放与方法参数
- 高级分析支持交互项、校正方式、诊断项
- 高级分析默认显示诊断标签页入口
- 高级分析允许用户手动覆盖系统推荐方法

### 3.5 结果详情页

#### 页面态

- 成功结果态
- 部分成功态
- 失败留痕态
- 高敏导出待审批态
- 派生变量可发布态

#### 高保真建议

- 页面顶部固定结果上下文带
- 摘要卡 + `ExecutionManifest` 摘要卡并列
- 图表区支持主图与备选图切换

#### 页面间跳转

- 来源可为：
  快速分析、高级分析、任务中心、下载中心
- 去向可为：
  图表工作台、派生变量发布、新版本详情、下载中心

### 3.6 图表工作台

#### 页面态

- 默认模板态
- 自定义编辑态
- 导出预览态
- 导出成功态

#### 高保真建议

- 预览区最大化
- 属性面板采用分组折叠
- 导出预览单独强化

### 3.7 问卷预览与答卷页

#### 页面态

- 项目内登录填写态
- 外部匿名填写态
- 填写中暂存态
- 跳题逻辑触发态
- 提交成功态
- 映射待确认态

#### 高保真建议

- 外部匿名问卷减少平台后台元素
- 顶部只保留品牌/项目最小识别信息
- 移动端优先单列布局

#### 外部匿名填写高保真说明

- 去掉后台左侧导航和复杂全局栏
- 只保留品牌标识、问卷标题、进度和必要隐私提示
- 提交成功页应明确“已完成提交”与后续动作
- 若进入 `pending_match`，不对外显示内部映射细节，只显示“已接收，待系统匹配”

### 3.8 问卷历史导入页

#### 页面态

- 文件待上传态
- 模板匹配态
- 编号映射态
- 异常清单态
- 导入完成态

#### 高保真建议

- 中央主区域以批次处理流程呈现
- 异常项表格与主键映射卡片并列

#### 页面间跳转

- 来源可为：
  问卷模板页、问卷结果管理页、项目详情页
- 去向可为：
  导入批次结果页、问卷结果管理页

### 3.9 高敏导出审批相关页

#### 页面态

- 申请中
- 待审批
- 审批通过
- 审批拒绝
- 链接过期

#### 高保真建议

- 审批状态使用醒目状态标签
- 过期和拒绝态强调后续动作

#### 审批态细化

- `pending_approval`
  显示申请信息、审批人、预计有效期
- `approved`
  显示下载入口、剩余次数、到期时间
- `rejected`
  显示拒绝原因、重新申请入口
- `expired`
  显示失效提示、重新发起申请入口

### 3.10 平台级受限分析空间

#### 页面态

- 初始警示态
- 已选跨租户数据态
- 聚合分析结果态
- 明细导出申请态

#### 高保真建议

- 始终显示强警示页头
- 视觉上与普通项目空间明显区隔
- 默认聚合结果卡片更突出

#### 页面间跳转

- 入口仅来自平台管理页
- 结果页不可直接跳回普通项目分析工作台
- 导出必须跳转到受限空间内的审批链路

## 4. 页面高保真说明

### 4.1 配色

- 普通项目空间沿用主品牌深青蓝体系
- 受限分析空间增加更明显的暖色警示条

### 4.2 密度模式

- 工作台：`comfortable`
- 分析工作台：`compact`
- 结果宽表 / 审计：`dense`

### 4.3 统计语义色

- 正相关：蓝绿系正向色
- 负相关：暖色负向色
- 显著：加深主色或深灰强调
- 不显著：中性色弱化

### 4.4 图表默认主图映射

- 差异分析：误差线图
- 描述统计：箱线图或柱形图
- 相关分析：热力图
- 回归分析：散点回归图 + 森林图
- 因子分析：载荷热图
- 问卷列联：残差热图

### 4.5 关键组件状态建议

- 变量投放器：
  `empty / filled / invalid / disabled_by_brand_switch`
- 字段树筛选器：
  `default / searching / filtered / empty_result`
- 结果摘要卡：
  `loading / success / partial_success / failed`
- 审批状态标签：
  `pending / approved / rejected / expired`
- 问卷渲染器：
  `idle / filling / saving / submitted / invalid`
- Data Explorer 明细表：
  `idle / loading / loaded / empty / forbidden`
- Data Explorer 图表预览卡：
  `idle / rendering / rendered / no_field_selected`
