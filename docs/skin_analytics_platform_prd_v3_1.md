# 企业级皮肤大数据管理与智能分析平台 PRD v3.1

## 0. 文档说明

### 0.1 文档目标

本版本 PRD 用于产品、设计、架构、研发、测试的联合评审，目标不是描述“想做什么”，而是明确：

- 谁在什么场景下使用系统
- 用户在每个页面如何操作
- 系统在每一步给出什么反馈
- 统计分析按什么规则自动决策
- UI 和 UX 需要达到什么标准
- MVP 与后续版本的边界在哪里

### 0.1.1 本版新增决策

本版在 v2 基础上，将以下关键业务规则正式写入 PRD：

- `Brand` 明确定义为品牌
- 一个项目允许绑定多个品牌
- 普通用户可在授权品牌范围内切换并分别分析，不允许跨品牌混合分析
- 跨品牌混合分析仅允许平台超级管理员在受限平台级分析空间执行，并允许跨租户
- 数据导入同时支持持续批次导入与一次性课题导入
- 分析结果可回写为派生变量，并发布为新数据版本
- 问卷需要支持量表计分、反向题、多选题、跳题逻辑
- 问卷支持外部填写
- 问卷结果同时支持在线实时答卷与批量导入历史答卷
- 问卷计分规则由超级管理员和品牌配置管理员维护，项目可复制后改写
- 高敏导出采用单人审批，审批人为租户管理员

### 0.2 产品定位

构建一个面向企业客户、兼顾业务研究和科研分析的多租户皮肤大数据管理与智能分析平台。平台既要满足企业级权限治理、审计追踪、数据安全要求，也要满足研究分析中的统计严谨性、图表导出质量和结果可复现要求。

### 0.3 本期范围

本期重点覆盖以下能力：

- 多租户组织管理与项目空间隔离
- 品牌/项目级指标配置与问卷配置
- 结构化数据接入、清洗、预处理
- 分组筛选与比较分析
- 描述性统计、差异性分析、相关性分析、多因素分析
- 因子分析、聚类分析、问卷列联分析
- 科研级图表在线预览与高质量导出
- 方法说明、过程留痕、结果下载、审计追溯
- 分阶段建设，但保留完整目标架构设计

### 0.4 暂不纳入本期范围

- 图像识别、皮肤照片 AI 诊断、病灶自动分割
- 时序预测、因果推断、深度学习建模
- 复杂纵向混合模型、结构方程模型、生存分析
- 对外开放 API 商业化能力
- 完整 BI 看板搭建器

## 1. 产品目标与成功标准

### 1.1 核心目标

- 让租户在 30 分钟内完成一次从上传到导出结果的完整分析流程
- 保证任意租户之间数据、文件、缓存、日志不可串读
- 让非统计专业用户也能在系统引导下完成正确的方法选择
- 保证导出的图表和结果可以直接用于内部汇报或科研投稿初稿

### 1.2 成功指标

- 首次上传成功率 >= 90%
- 首次分析任务完成率 >= 85%
- 关键分析任务失败率 <= 2%
- 审计日志覆盖率 = 100%
- 导出文件可复现率 = 100%

### 1.3 产品原则

- 安全优先：任何跨租户、跨项目越权默认禁止
- 过程透明：系统必须说明“做了什么”和“为什么这样做”
- 结果可追溯：所有分析必须可回放、可下载、可审计
- 操作低学习成本：以向导、预览、即时反馈替代隐藏式配置
- 科研严谨：方法选择、前提检验、事后比较、效应量输出形成闭环

## 2. 角色定义与权限矩阵

### 2.1 角色定义

- 平台超级管理员：管理平台级租户、套餐、实例配置、安全策略、系统审计
- 租户管理员：管理本租户内用户、项目、权限、数据保留策略、下载策略
- 项目管理员：管理项目成员、数据集可见性、分析模板、共享结果
- 品牌配置管理员：管理品牌指标可用性、问卷模板、字段映射和分析模板
- 数据上传员：上传文件、映射字段、执行预处理、提交数据版本
- 分析师/研究员：创建分组、运行统计分析、配置图表、导出文件
- 业务查看者：查看结果、浏览图表、下载授权范围内文件
- 审计/合规人员：查看登录日志、数据访问日志、导出日志、分析过程日志

### 2.3 权限规则

- 所有操作必须同时满足租户权限和项目权限
- 用户进入页面前先校验租户上下文，再校验项目权限
- 文件下载、过程文件下载、原始数据下载必须单独授权
- 审计人员默认只读，不得修改业务数据
- 分析模板、图表模板、导出模板默认归属项目，是否可跨项目共享由项目管理员控制
- 普通用户定义为租户内或项目内被授权用户，包括项目管理员、品牌配置管理员、数据上传员、分析师、业务查看者
- 普通用户在多品牌项目中可以在授权品牌之间切换并分别分析，但不得执行跨品牌混合分析
- 平台超级管理员可进入平台级受限分析空间执行跨品牌、跨租户混合分析，该空间的所有操作必须开启强审计
- 高敏导出审批人为租户管理员，审批通过后才允许生成正式导出包

## 3. 关键业务对象

- Tenant：租户，平台最小隔离单元
- OrganizationUser：组织用户，绑定一个租户，可属于多个项目
- Project：项目/课题/数据空间，业务协作最小管理单元
- Brand：品牌/客户侧业务配置实体
- BrandConfigVersion：品牌配置版本
- InstrumentFamily：仪器或算法指标家族
- BodySite：测试部位/测量点
- Dataset：数据集，表示一份业务数据实体
- DatasetVersion：数据版本，表示同一数据集的某次导入结果
- ImportBatch：导入批次
- VariableMeta：字段元数据，包含字段名、类型、标签、单位、编码规则
- MetricCatalog：统一指标目录
- MetricCatalogVersion：统一指标目录版本
- MetricAlias：原始字段名和标准指标映射
- BrandMetricAvailability：品牌-指标-部位可用性矩阵
- QuestionnaireTemplate：问卷模板
- QuestionnaireTemplateVersion：问卷模板版本
- QuestionnaireQuestion：问卷题目
- QuestionnaireOption：问卷选项
- Subject：样本主体，例如受试者、客户、患者
- SubjectIdentifier：外部编号映射实体
- VisitRecord：时间点记录，用于支持一人多次记录
- CohortRule：分组和筛选规则定义
- AnalysisJob：分析任务，包含方法、参数、输入、状态
- AnalysisResult：分析结果，包含统计量、显著性、效应量、图表、解释
- AnalysisTemplate：分析模板
- NarrativeTemplate：结果叙述模板
- DerivedVariableSet：派生变量集合
- ExecutionManifest：不可变执行快照
- VersionBundle：品牌维度原子版本包
- ExportApprovalRequest：高敏导出审批单
- ChartConfig：图表配置
- ExportPackage：导出包，含结果文件、图像文件、方法快照、过程文件
- AuditLog：审计日志

## 5. 端到端用户链路

### 5.1 链路 A：租户开通与初始化

适用角色：平台超级管理员、租户管理员

前置条件：

- 平台已具备可用实例资源
- 租户商务信息、套餐、隔离模式已确认

主链路：

1. 平台超级管理员创建租户
2. 选择隔离模式：共享集群隔离 / 独享实例
3. 初始化租户管理员账号
4. 租户管理员首次登录并修改密码
5. 配置组织信息、部门结构、项目分类、下载策略
6. 邀请项目管理员和成员

系统反馈：

- 创建成功后生成租户 ID、默认空间、审计记录
- 首次登录必须完成密码修改和安全设置
- 若租户未配置默认项目模板，则进入引导页

异常分支：

- 租户开通失败：提示具体失败步骤并保留回滚日志
- 管理员邀请失败：展示邮箱/手机号格式错误或重复绑定原因

### 5.1.1 补充链路：项目初始化与模板绑定

适用角色：租户管理员、项目管理员、品牌配置管理员

前置条件：

- 项目已创建
- 已存在至少一个可发布的品牌配置版本

主链路：

1. 项目管理员创建项目
2. 选择项目允许绑定的一个或多个品牌
3. 为每个品牌绑定品牌配置版本、指标目录版本、问卷模板版本
4. 配置项目级导出策略和高敏导出审批规则
5. 配置项目默认数据导入模式：持续批次 / 一次性课题
6. 配置项目默认分析模板和图表模板
7. 项目进入可上传、可分析状态

系统反馈：

- 系统展示项目当前绑定的品牌清单及各自版本号
- 若品牌模板与问卷模板版本不兼容，阻断发布并提示冲突
- 若项目尚未绑定任何品牌，不允许进入正式上传和分析链路

异常分支：

- 品牌版本被废弃：允许沿用历史版本，但禁止新绑定
- 项目切换模板版本：必须提示受影响的数据版本、分析模板和历史结果范围

### 5.1.2 补充链路：角色首登路由

适用角色：品牌配置管理员、数据上传员、分析师、项目管理员

规则：

- 品牌配置管理员首登默认进入品牌配置页
- 数据上传员首登默认进入最近项目的数据集列表页；若无项目，则进入项目申请引导页
- 分析师首登默认进入最近项目的分析工作台；若无已发布数据版本，则引导到数据集页
- 项目管理员首登默认进入项目初始化页，优先完成品牌模板绑定

首登 CTA：

- 无项目：申请加入项目 / 创建项目
- 项目未绑定品牌：进入项目初始化与模板绑定
- 无已发布数据版本：进入上传向导
- 无分析模板：进入快速分析入口

### 5.2 链路 B：数据接入与入库

适用角色：数据上传员、项目管理员

前置条件：

- 已选择项目
- 用户拥有上传权限

主链路：

1. 进入数据集列表页，点击“新建数据集”
2. 填写数据集名称、描述、样本主键规则、数据类型
3. 上传 Excel、CSV 或固定格式文本文件
4. 系统识别编码、工作表、字段名、字段类型
5. 用户确认字段映射、主键字段、时间字段、分组字段
6. 系统执行模板校验、重复校验、缺失值扫描、异常值扫描
7. 用户在预处理页选择处理策略或仅标注不处理
8. 系统生成数据版本并入库

系统反馈：

- 上传后立即显示解析进度
- 校验结果以“错误 / 警告 / 提示”三级展示
- 主键冲突和字段缺失属于阻断错误
- 缺失值和异常值默认属于警告，可继续但必须留痕

异常分支：

- 编码识别失败：允许用户手动切换 GBK / UTF-8 / UTF-8 BOM
- 字段类型识别冲突：允许用户手动修正类型并重跑校验
- 文件模板不合法：提供失败行下载与模板示例

### 5.3 链路 C：分组构建与样本筛选

适用角色：分析师、研究员

前置条件：

- 数据版本已发布
- 字段元数据已确认

主链路：

1. 进入分组构建页
2. 选择分析基准数据版本
3. 按变量拖拽或点选生成筛选条件
4. 设置单条件或多条件逻辑组合
5. 配置分组合并规则，例如“中度+重度”
6. 实时查看样本量变化、缺失情况、组间分布
7. 保存为分组规则或直接进入分析

系统反馈：

- 右侧固定显示当前样本数、各组样本数、被排除样本数
- 当某组样本量过低时，系统弹出风险提示但不强制阻断
- 合并规则必须显式显示来源组与目标组

异常分支：

- 条件互斥导致样本为 0：禁止保存并给出最近冲突条件提示
- 分组后某组样本量不足最小分析要求：提示用户调整规则

### 5.4 链路 D：统计分析与图表输出

适用角色：分析师、研究员、项目管理员

前置条件：

- 已选定数据版本和分组规则
- 用户拥有分析权限

主链路：

1. 在分析工作台选择分析类型
2. 指定因变量、自变量、分组变量、协变量
3. 系统根据变量类型自动推荐可用方法
4. 用户确认方法与高级参数
5. 提交分析任务
6. 系统异步执行前提检验、主体分析、效应量计算、图表生成
7. 用户查看结果摘要、详细结果、图表和方法说明
8. 用户保存结果、导出文件或复制为模板

系统反馈：

- 任务进入队列后展示预计等待时间和状态
- 分析结果页顶部固定展示“品牌配置版本 + 数据版本 + 分组规则 + 方法 + 样本量”
- 若系统自动替换推荐方法，必须明确说明原因

异常分支：

- 方法前提不满足：允许用户接受系统推荐或强制继续并留痕
- 某变量缺失过多：阻断分析并提示变量质量问题
- 任务超时：保留输入参数，允许重试

### 5.5 链路 E：结果追溯与审计

适用角色：租户管理员、审计人员、业务查看者

主链路：

1. 进入结果详情或下载中心
2. 查看分析输入、方法、参数、时间、执行人
3. 下载结果文件、图像、过程文件、方法说明
4. 审计人员进入日志页查看谁在何时访问和导出了什么

系统反馈：

- 每次下载必须产生一条审计日志
- 每个结果页面都能回跳到原始分析任务
- 下载的 ZIP 包中必须包含 `readme` 或方法说明文件
- 结果页必须展示 `ExecutionManifest` 摘要，包括配置版本、模板版本、引擎版本和数据版本

## 6. 详细功能需求

### 6.1 租户与组织管理

#### 功能目标

支持多租户环境下的企业组织、项目空间、角色权限、安全策略管理。

#### 核心要求

- 一个用户账号只能归属一个租户，不支持跨租户共用
- 一个租户下可创建多个项目，项目之间默认隔离
- 租户管理员可配置项目成员、数据权限、导出权限
- 支持角色模板和自定义细粒度权限
- 支持用户停用、强制下线、密码重置、MFA 预留

#### 页面操作要求

- 用户管理页支持列表、搜索、筛选、批量启停用
- 角色配置页支持权限树配置和权限差异预览
- 项目成员页支持按角色批量邀请

### 6.2 数据上传与导入

#### 支持文件类型

- Excel：`.xlsx`、`.xls`
- CSV：支持逗号分隔、制表符分隔
- 文本：固定模板格式，用于预定义组间比较输入

#### 上传规则

- 单文件大小默认上限 500 MB，可按租户策略配置
- 大文件采用分片上传
- 上传后先进入临时解析区，不直接入库
- 系统自动识别编码；若识别置信度不足，要求用户确认
- 支持中文字段名、中文选项值、中文文件名

#### 字段识别规则

- 自动识别字段类型：数值型、分类型、布尔型、日期型、文本型
- 自动识别主键候选字段
- 自动识别时间点字段和可能的分组字段
- 自动识别缺失编码，如空白、NA、N/A、NULL、-9999

#### 验收要求

- 重复主键必须明确提示重复数量和示例记录
- 非法数值必须提示具体行号和字段名
- 无法识别的字段类型必须允许用户手工修正

### 6.2.1 品牌/项目指标配置

#### 目标

支持不同品牌、不同项目使用不同的指标集合、问卷集合和测量部位，不将全量指标大表直接暴露给所有租户或项目。

#### 核心要求

- 品牌下可配置可用仪器家族
- 品牌下可配置可用指标
- 品牌下可配置部位/测量点
- 品牌下可配置是否启用问卷及具体题项
- 一个项目可绑定一个或多个品牌配置模板
- 上传数据时根据项目绑定的品牌配置自动校验字段合法性

#### 典型配置粒度

- 品牌
- 仪器
- 部位
- 指标
- 是否必填
- 是否展示给分析师
- 是否允许导出

### 6.2.2 统一指标目录与字段映射

#### 目标

将原始 Excel 列名和系统内部标准指标解耦，支撑宽表治理、模板复用和后续分析扩展。

#### 核心要求

- 原始字段名不可直接作为唯一业务主字段
- 每个字段必须可映射到标准指标目录
- 支持一对一映射、一对多映射、别名映射
- 支持字段分类：基础字段、问卷字段、仪器字段、医生评估字段、派生字段
- 支持字段打标签：仪器家族、部位、变量类型、是否可分析、是否默认展示

### 6.2.3 系统默认配置与二次编辑

#### 目标

系统首次上线后即具备可用的品牌配置、指标目录、问卷模板和分析模板，不要求业务用户从零开始手工配置。

#### 默认配置来源

- 系统内置默认品牌配置种子数据
- 系统内置默认指标目录种子数据
- 系统内置默认问卷模板种子数据
- 系统内置默认分析模板和图表模板
- 支持通过初始化脚本导入现有 Excel/Word 配置文件生成首版配置

#### 应用加载后的默认显示规则

- 用户进入项目后，默认只显示当前项目绑定品牌下“已启用且可分析”的指标
- 默认不展示全量原始字段大表中的全部 1400+ 列
- 指标目录页默认展示标准指标视图，可切换查看原始字段映射视图
- 分析工作台默认按品牌配置过滤变量候选项
- 问卷题项默认按已绑定问卷模板展示

#### 可编辑规则

- 系统级默认配置可由平台管理员维护
- 租户和项目不可直接覆盖系统基线配置本体，默认采用“复制为租户版/项目版后编辑”
- 已被项目使用的配置修改必须版本化，不允许静默覆盖
- 指标启停用、部位启停用、问卷题项启停用都必须记录变更历史
- 问卷计分规则由超级管理员和品牌配置管理员维护为系统/品牌基线规则
- 项目允许复制问卷计分规则并生成项目级改写版本
- 项目级改写仅作用于当前项目，不反向覆盖品牌基线规则

#### 推荐编辑入口

- 品牌配置页：编辑品牌可用指标、仪器、部位、问卷模板绑定
- 指标目录页：编辑标准指标、别名、标签、变量类型、默认展示状态
- 问卷模板页：编辑题目、选项、题型、分组标签
- 项目设置页：选择当前项目绑定哪一版品牌配置模板

#### 推荐版本策略

- `System Default`：平台内置基线版本
- `Tenant Override`：租户级复制版
- `Project Override`：项目级复制版

#### 配置生命周期

- `Draft`：编辑中，不可用于正式项目绑定
- `Published`：已发布，可绑定到项目
- `Deprecated`：不再允许新绑定，但历史结果继续可追溯

#### 风险控制

- 若配置变更会影响历史分析结果，系统必须提示影响范围
- 历史分析任务默认继续绑定原配置版本，不随新配置自动漂移

### 6.2.4 主键映射与多源整合规则

#### 目标

解决仪器数据、微生态数据、问卷数据、医生评估数据来自不同文件和不同编号体系时的统一识别、批次管理和版本整合问题。

#### 主键设计原则

- 系统内部永远使用 `subject_id` 作为受试者主键
- 系统内部永远使用 `visit_id` 作为同一受试者的单次采集/单次批次记录主键
- 外部编号只作为映射锚点，不直接充当系统内部唯一主键

#### 当前默认外部编号策略

结合当前样本数据，外部编号优先级定义如下：

- `微生态编号`：首选外部唯一编号
- `编号：RD`：高频业务编号/展示编号/问卷对接编号
- `您的志愿者编号：RD-...`：问卷侧外部编号，默认映射到 `RD`

原因：

- 当前主数据中 `微生态编号` 唯一性高，更适合作为主锚点
- `RD` 在现有数据中存在重复，更适合作为业务展示编号和跨文件辅助映射键

#### 核心实体

- `Subject`
- `SubjectIdentifier`
- `ImportBatch`
- `VisitRecord`

#### 识别与整合规则

1. 若文件中存在 `微生态编号`，优先以其作为外部唯一编号建立或匹配 `Subject`
2. 若缺失 `微生态编号`，但存在 `RD` 或问卷侧 `RD-` 编号，则进入编号映射流程
3. 问卷侧 `RD-` 默认先标准化为 `RD`
4. 同一项目内，系统允许一个 `Subject` 绑定多个外部编号
5. 禁止使用姓名、年龄、城市等弱特征自动合并受试者
6. 当无法自动匹配时，允许人工映射，但必须留痕并记录映射人、时间、原因

#### 批次与时间点规则

- 若业务文件提供时间点字段，则按业务时间点生成 `visit_id`
- 若业务文件未提供时间点字段，则系统按导入批次自动生成 `visit_id`
- 持续批次导入时，同一 `subject_id` 可存在多个 `visit_id`
- 一次性课题导入时，可仅生成一个默认 `visit_id`

#### 多源导入模式

- `Append`：新增样本或新增时间点
- `Merge`：向既有样本补充新字段
- `Replace`：替换某一导入批次内的错误数据
- `Patch`：局部修订已存在记录

#### 冲突处理优先级

- 完全匹配的唯一外部编号
- 已存在的人工确认映射
- 用户本次手动指定映射
- 其余情况一律不自动合并

#### 多品牌项目规则

- 项目可绑定多个品牌
- 普通用户可在授权品牌间切换并分别分析
- 普通用户不得跨品牌混合分析
- 超级管理员可在平台级受限分析空间执行跨品牌、跨租户混合分析

#### 审计要求

- 所有人工映射、批次替换、局部修订、批次回滚必须写审计日志
- 数据整合结果必须可回溯到源文件、导入批次、映射规则和执行人

### 6.2.5 VersionBundle 与原子绑定规则

#### 目标

在多品牌项目下，确保分析任务绑定的是单品牌的一组一致性版本，而不是零散配置组合。

#### 定义

`VersionBundle` 由以下对象组成：

- `brand_config_version`
- `metric_catalog_version`
- `questionnaire_template_version`

#### 规则

- 一个分析任务必须绑定一个品牌的一个 `VersionBundle`
- 一个项目可同时绑定多个品牌的多个 `VersionBundle`
- 用户在切换品牌时，实际切换的是该品牌对应的 `VersionBundle`
- 版本切换必须成组生效，不允许只切换其中一个子版本
- 历史分析继续绑定旧 `VersionBundle`，不随项目最新配置漂移

### 6.3 数据预处理

#### 目标

在不破坏原始数据的前提下，生成可分析的数据版本，并完整保留处理轨迹。

#### 预处理能力

- 缺失值统计：按字段、按组、按样本维度展示
- 缺失值处理：仅标注、删除记录、删除字段、均值填补、中位数填补、众数填补
- 异常值识别：IQR 法、Z-score 法，默认仅标注不自动删除
- 单位归一：同一字段多单位时允许统一换算
- 类别标准化：如“男/女”、“M/F”、“male/female”归一
- 字段重命名、标签补充、字段描述维护

#### 处理原则

- 原始文件不可覆盖
- 每次处理产生新的数据版本
- 每个处理动作都记录执行人、时间、规则、影响记录数
- 对可能改变统计结论的处理动作必须二次确认

#### 数据版本状态

- `Draft`：上传或预处理中，尚未进入正式分析
- `Published`：正式发布，可用于正式分析
- `Frozen`：冻结版，仅允许查看和复现
- `Archived`：归档版，不再参与日常分析

#### 数据版本规则

- 普通分析默认只能基于 `Published` 或 `Frozen` 版本执行
- 所有新派生变量发布后，必须生成新的 `DatasetVersion`
- 历史分析结果继续绑定原始 `DatasetVersion`，不随新版本漂移

### 6.4 分组与筛选引擎

#### 目标

让用户可以通过低门槛的可视化规则构建方式，得到严格可复现的比较组或分析样本。

#### 支持的条件类型

- 数值条件：`=`, `!=`, `>`, `>=`, `<`, `<=`, `between`
- 类别条件：单选、多选、排除、组合
- 四分位条件：Q1、Q2、Q3、Q4
- 文本条件：包含、精确匹配、前缀、后缀
- 缺失条件：是否缺失

#### 支持的逻辑结构

- 单条件
- 多条件 AND
- 多条件 OR
- 条件组嵌套
- 条件取反

#### 支持的分组合并

- 原始分组映射为目标分组
- 多个原始选项合并为一个目标组
- 支持保存合并规则模板
- 合并后实时展示新旧组样本量变化

#### UX 要求

- 条件构建区左侧为字段树，中间为规则画布，右侧为结果预览
- 每次条件变更后 1 秒内返回样本量预览
- 风险提醒不阻断主流程，但必须醒目标识

### 6.5 分析工作台

#### 目标

让用户在一个统一工作台中完成变量选择、方法推荐、参数确认、任务执行、结果查看。

#### 工作台结构

- 顶部：当前数据版本、当前分组规则、当前样本量
- 左侧：分析类型与变量配置区
- 中间：方法推荐、参数配置、前提提示
- 右侧：结果预览、图表预览、方法说明

#### 核心能力

- 根据变量类型自动推荐可用分析方法
- 支持手动覆盖推荐方法，但必须记录原因
- 支持保存分析模板
- 支持任务队列、历史任务、重跑任务、复制任务

### 6.5.2 快速分析与高级分析双模式

#### 快速分析

适用用户：

- 非统计专业用户
- 需要快速得到标准结论的业务用户

特点：

- 仅暴露少量输入项
- 系统自动推荐方法
- 高级参数默认隐藏
- 默认输出主图、主表、简版解释

#### 高级分析

适用用户：

- 分析师
- 研究员
- 高级用户

特点：

- 暴露完整方法与高级参数
- 支持交互项、校正方式、诊断项、聚类参数等
- 支持手动覆盖系统推荐方法，并强制留痕

#### 默认入口

- 若用户首次进入分析工作台且无保存模板，默认进入快速分析
- 高级分析为显式切换入口，不与快速分析混排

#### 方法配置区要求

- 必填项缺失时禁止提交
- 方法说明默认展开简版，可点击查看详细版
- 高级参数默认折叠，避免压垮初级用户

### 6.5.1 派生变量回写与新版本生成规则

#### 目标

将分析结果中可复用的结构化产出保存为派生变量，并以新数据版本形式进入后续分析闭环。

#### 可回写的派生变量类型

- 因子得分
- 聚类类别标签
- 回归预测值
- 回归残差
- 问卷总分/维度分
- 标准化值、分层标签、规则打标结果

#### 默认发布方式

- 分析完成后先生成 `DerivedVariableSet`
- 用户确认后，将派生变量发布为新的 `DatasetVersion`
- 不允许直接覆盖原始数据版本
- 当前阶段不要求项目管理员再次确认，分析师可直接发布

#### 发布规则

- 新版本必须记录来源 `AnalysisJob`
- 每个派生变量必须记录：来源方法、来源字段、算法版本、生成时间、生成用户
- 派生变量进入新版本后，默认可作为后续分析变量使用
- 历史数据版本不可被回写修改

#### 命名与可见性规则

- 派生变量默认带方法前缀和版本信息
- 用户可在发布前修改显示名称、标签、变量说明
- 派生变量默认仅在当前项目内可见

#### 审计与追溯

- 每次派生变量发布必须生成审计记录
- 结果页必须可回跳到对应的 `AnalysisJob` 和 `DerivedVariableSet`
- 新版本详情页必须显示其由哪一个旧版本派生而来

#### 版本正确性规则

- 派生变量发布后必须生成字段差异摘要
- 同一 `AnalysisJob` 重复发布时，系统必须提示是否生成新版本或复用已发布结果
- 冻结版之上不允许直接回写，只能从冻结版复制生成新版本

#### 后续兼容设计

- 后续版本可扩展“派生变量草稿区”
- 后续版本可支持批量勾选哪些派生变量发布
- 后续版本可支持派生变量再次冻结或归档

### 6.6 图表编辑与导出

#### 目标

让用户在不依赖外部绘图工具的情况下完成科研级图表配置与导出。

#### 基础图表类型

- 柱形图，支持误差棒、组间对比、显著性标注
- 箱型图，支持抖动散点、异常值标记

- 聚类树状图
- 饼图
- 森林图
- 方差解释率图
- 标准化残差热力图
- 堆叠条形图

#### 图表配置项

- 图标题、副标题
- 横纵坐标标题
- 字体大小、中英文字体
- 颜色主题、配色方案
- 标签显示规则
- 坐标轴范围、刻度、单位
- 图例位置

### 6.7 结果追溯与下载

#### 下载内容

- 结果摘要表
- 明细统计表
- 事后比较结果
- 图像文件
- 方法说明文件
- 参数快照文件
- 过程日志文件

#### 下载规则

### 6.7.1 高敏导出审批生命周期

- `pending_approval`
- `approved`

## 8. 全局 UX 与交互原则

### 8.1 设计原则

- 先上下文，后操作：在任何关键页面，先让用户确认当前租户、项目、数据版本，再开始操作
- 先预览，后提交：所有可能改变数据或统计结果的动作都先展示影响范围
- 先解释，后阻断：方法不满足前提时，先明确解释，再决定是阻断还是允许强制继续
- 先默认安全，后高级开放：默认采用最稳妥选项，高级参数折叠到高级设置

### 8.2 空状态

- 无项目时：提示联系租户管理员开通权限
- 无数据集时：引导上传第一个数据集
- 无分析结果时：引导进入分析工作台
- 无权限时：明确说明缺少哪类权限，不展示“假空页”

### 8.3 加载状态

- 解析上传文件时展示分阶段进度
- 分析任务采用可感知队列状态
- 图表加载超时后提示稍后重试，不让页面白屏

### 8.4 错误状态

- 错误文案必须说明原因、影响、建议动作
- 上传失败必须支持失败行下载
- 分析失败必须保留输入参数与日志编号
- 品牌切换导致模板失效时，必须说明失效原因与替代动作
- 主键映射冲突、派生变量冲突、高敏导出审批失败必须有专属错误态

### 8.5 表单与校验

- 实时校验字段格式
- 关键提交按钮仅在必填完整时可点
- 风险参数变更必须二次确认

### 8.6 通知与消息

- 成功提示使用轻反馈，不中断流程
- 失败提示要定位到动作和对象
- 长任务完成支持站内通知

### 8.7 表格与图表交互

- 表格默认冻结关键列
- 图表鼠标悬浮展示精确数值
- 图表和表格支持联动高亮
- 导出前允许预览最终风格

### 8.8 可用性要求

- 中文界面为主，英文术语可辅助显示
- 支持 1440 宽度桌面优先，同时适配 1280 宽屏
- 核心分析流程不依赖右键、悬浮才能发现关键功能

### 8.9 视觉设计方向

整体风格定义为“医疗科研感 + 企业分析平台感”，强调专业、克制、可信、精确，不采用泛互联网后台常见的高饱和渐变风格，也不采用消费级 App 的娱乐化视觉。

视觉关键词：

- 专业
- 洁净
- 冷静
- 数据可信
- 科研感
- 企业级秩序感

## 9. 统计分析方法规范

### 9.1 变量类型识别规则

- 连续型变量：年龄、评分、指标值、测量值
- 分类型变量：性别、分型、问卷选项、等级
- 序数变量：轻度/中度/重度、有序评分
- ID 字段：样本编号、记录编号，不参与统计
- 时间字段：采集时间、随访时间，用于筛选和后续扩展，不默认参与统计

### 9.2 通用分析前置规则

- 所有分析必须绑定一个数据版本
- 所有分析必须绑定一个品牌上下文；普通用户不得在单次任务中绑定多个品牌
- 跨品牌混合分析仅允许平台超级管理员执行
- 所有分析必须记录筛选规则或明确标注“全量样本”
- 分析前先检查有效样本量、缺失情况、变量类型兼容性
- 组间比较分析要求分组变量至少存在 2 个非空组
- 样本独立性默认由业务场景保证，用户提交任务时需显式确认
- 描述统计、差异分析、回归分析默认采用完整案例分析
- 相关矩阵默认采用成对删除缺失值，并展示有效样本量信息
- 当有效样本量小于系统最低阈值时，不阻断但必须高亮提示结果不稳定
- 所有结果默认展示置信水平 95%
- 每个 `AnalysisJob` 必须生成不可变 `ExecutionManifest`

### 9.3 频数分析与样本画像

#### 适用场景

- 样本背景描述
- 问卷单题分布
- 分类变量构成比统计
- 分品牌、分城市、分年龄组样本画像输出

#### 输入

- 一个或多个分类变量
- 可选分组变量

#### 输出

- 样本量 N
- 缺失值数与缺失率
- 频数
- 百分比
- 有效百分比
- 累计百分比
- 按组频数对比表

#### 图表

- 条形图
- 堆叠条形图
- 饼图
- 样本画像汇总表

### 9.4 描述性统计

#### 适用场景

- 对单个变量或按组变量进行概览

#### 输入

- 单个或多个连续型变量
- 可选分组变量

#### 输出

- 样本量 N
- 缺失值数与缺失率
- 均值、标准差
- 中位数、四分位数
- 最小值、最大值
- 偏度、峰度

#### 图表

- 柱形图
- 箱型图
- 直方图

### 9.5 差异性分析方法决策表

| 场景 | 输入条件 | 前提检验 | 推荐方法 | 主要输出 |
| --- | --- | --- | --- | --- |
| 两组连续变量比较 | 因变量连续，分组变量二分类 | Shapiro-Wilk，方差齐性 | 独立样本 t 检验 / Welch t 检验 / Mann-Whitney U | 均值差、p 值、置信区间、效应量 |
| 多组连续变量比较 | 因变量连续，分组变量多分类 | Shapiro-Wilk，方差齐性 | 单因素 ANOVA / Welch ANOVA / Kruskal-Wallis | F 或 H 值、p 值、效应量、事后比较 |
| 合并组比较 | 由多个原始类别合并而成 | 同上 | 按合并后分组自动选择 | 与普通分组一致 |

### 9.6 差异性分析执行规则

两组比较时：

- 若各组近似正态且方差齐，默认使用独立样本 t 检验
- 若各组近似正态但方差不齐，默认使用 Welch t 检验
- 若不满足正态性，默认推荐 Mann-Whitney U

多组比较时：

- 若各组近似正态且方差齐，默认使用单因素 ANOVA
- 若各组近似正态但方差不齐，默认使用 Welch ANOVA
- 若不满足正态性，默认使用 Kruskal-Wallis

用户手动覆盖系统推荐方法时：

- 系统必须记录“用户手动指定”
- 结果页必须显示“方法由用户强制选择”

### 9.7 差异性分析输出要求

- 前提检验结果表
- 主体检验统计量与 p 值
- 组均值或秩均值
- 效应量
- 置信区间
- 事后多重比较结果
- 显著性标注图
- 方法选择说明

### 9.8 事后比较与多重校正

- ANOVA 后默认支持 Tukey HSD
- Welch ANOVA 后默认支持 Games-Howell
- Kruskal-Wallis 后默认支持 Dunn 检验
- 多重比较校正默认使用 Holm 或 Bonferroni，允许在高级参数中切换

### 9.9 效应量规则

- 两组参数比较：Cohen's d，样本量较小时优先 Hedges' g
- ANOVA：Eta squared 或 Omega squared
- Kruskal-Wallis：Epsilon squared
- 相关性分析：相关系数本身即效应量，同时展示 p 值
- 回归分析：标准化回归系数、R²、调整后 R²

### 9.10 正态性与方差齐性检验规则

- 正态性默认使用 Shapiro-Wilk
- 方差齐性默认使用 Levene 检验
- 前提检验结果不直接等价于“分析不可用”，而是用于推荐更合适方法
- 当样本量很大时，页面提示用户结合图形分布与实际业务解释，不仅依赖 p 值

### 9.11 相关性分析

#### 输入

- 两个或多个连续型或序数型变量

#### 决策规则

- 连续型且接近正态时优先 Pearson
- 序数变量或非正态分布优先 Spearman
- 矩阵分析默认对所有变量两两计算

#### 输出

- 相关系数矩阵
- p 值矩阵
- 有效样本量矩阵
- 显著性星号标注
- 热图
- 可选聚类排序

### 9.12 回归分析

#### 支持范围

- 一元线性回归
- 多元线性回归
- 逐步回归

#### 输入

- 因变量 1 个，必须为连续型
- 自变量 1 个或多个，可为连续型或经编码后的类别变量
- 可选协变量

#### 诊断要求

- VIF 多重共线性
- 残差分布图
- 残差 vs 拟合值图
- 标准化系数
- R²、调整后 R²
- F 统计量与 p 值
- Cook's distance 预留为增强项

#### 逐步回归规则

- 默认采用双向逐步法
- 默认准则采用 AIC
- 高级参数允许切换为 BIC 或 p 值阈值策略
- 系统必须输出变量进入和移出每一步骤的过程表

### 9.13 多因素分析

#### 支持范围

- 普通线性模型
- 广义线性模型 GLM

#### 当前优先实现

- 当因变量明显偏态且为正值型时，支持 `Gamma + log link`
- 支持分类自变量、连续自变量、交互项
- 支持参考组设置

#### 输出

- 模型类型
- 分布与 link 函数
- 样本量
- 拟合优度
- RMSE 或等价误差指标
- 系数表
- 95% 置信区间
- 显著性标记
- 森林图
- 自动文字解释

### 9.14 因子分析与聚类分析

#### 支持范围

- 因子分析
- 基于因子得分的聚类分析

#### 因子分析输出

- 分析样本量
- 参与指标数量
- KMO 检验
- Bartlett 球形检验
- 因子载荷矩阵
- 方差解释率
- 因子命名说明

#### 聚类分析输出

- 聚类数量
- 各类样本分布
- 聚类可视化
- 各类高低特征
- 基于 Z-score 的亚型画像
- 亚型命名与自动描述

### 9.15 问卷列联与分类差异分析

#### 适用场景

- 问卷题项选项分布比较
- 不同城市、年龄组、品牌组之间的分类变量比较

#### 支持方法

- 列联表
- 卡方检验
- 标准化残差分析

#### 输出

- 列联表
- 卡方统计量
- p 值
- 标准化残差矩阵
- 残差热力图
- 结构饼图
- 堆叠条形图

### 9.16 相关性分析补充规则

- 支持按年龄组、城市、品牌等维度进行分层相关分析
- 支持设置相关系数阈值，仅在结果摘要中展示绝对值超过阈值的关系
- 热图支持全量矩阵与筛选矩阵两种模式

### 9.17 热图聚类排序

- 当用户启用聚类排序时，默认对变量做标准化
- 默认距离度量为欧氏距离
- 默认层次聚类方法为 Ward 或 Average，可配置
- 输出聚类树状图与排序后的热图

### 9.18 分析结果解释要求

- 每个结果页必须包含“统计结论”和“方法说明”两个区块
- 系统生成的文字结论只能作为辅助摘要，不能替代原始统计表
- 当结果不显著时，仍需完整展示统计量和效应量
- 多因素、聚类、问卷列联结果应支持结构化叙述模板

## 11. 多租户隔离与安全要求

### 11.1 隔离原则

建议采用“控制面共享、数据面隔离”的架构：

- 租户级数据库或 Schema 隔离
- 当前阶段采用租户级本地文件目录隔离
- 未来阶段可扩展为租户级对象存储路径隔离
- 租户级缓存命名空间隔离
- 租户级导出文件路径隔离
- 服务层强制租户上下文注入
- 租户级任务队列命名空间隔离
- 租户级分析执行临时目录隔离
- 租户级结果缓存和中间产物隔离

#### 平台级受限分析空间

- 仅平台超级管理员可进入
- 该空间为独立执行域，不复用普通项目计算上下文
- 默认仅允许输出聚合结果
- 明细导出必须额外审批
- 该空间产出的派生变量不得直接回写普通项目版本
- 该空间的模板、导出包和审计日志与普通项目空间分开管理

### 11.2 安全要求

- 全链路 HTTPS
- 静态数据加密
- 敏感文件病毒扫描
- 最小权限原则
- 登录日志、访问日志、导出日志全量记录
- 高风险下载支持二次确认和审批预留
- 高敏导出采用单人审批，审批角色为租户管理员
- 导出链接默认短时效签名
- 高敏导出支持水印、脱敏和到期回收
- 默认禁止申请人与审批人为同一人

### 11.3 审计要求

- 上传、删除、预处理、分析、导出都必须写审计日志
- 审计日志不可由业务用户删除
- 日志字段至少包含用户、时间、IP、项目、对象、动作、结果
- 人工主键映射、模板切换、派生变量发布、跨品牌混合分析必须单独高亮审计

## 12. 非功能需求

### 12.1 性能

- 100 MB 以内文件上传后 60 秒内完成结构化校验
- 常规分析任务在 30 秒内返回结果
- 重型分析任务采用异步队列，页面可离开
- 100 MB 到 500 MB 文件默认进入异步解析与校验队列
- 任务提交必须具备幂等键，防止重复提交

### 12.2 可靠性

- 任务失败可重试
- 导出任务失败可重跑
- 关键任务需具备幂等设计
- 统一任务状态机：`draft -> queued -> running -> partial_success -> succeeded / failed / cancelled / expired`
- 重试次数、超时阈值和并发上限按租户策略配置
- 部署或服务重启后，运行中任务必须支持恢复或显式回收
- `ExecutionManifest` 必须作为结果复现的基础元数据快照
- 失败任务的中间产物必须按 TTL 策略清理
- 队列调度需支持租户级并发配额与公平性

### 12.3 可维护性

- 分析方法和 UI 配置分层设计
- 图表预览和科研导出分离实现
- 方法参数和结果结构化存储，避免写死在页面逻辑里
- 数据版本、配置版本、分析模板版本和执行快照必须解耦存储

### 12.4 恢复目标与保留策略

- 定义 `RPO` 与 `RTO`
- 定义数据版本备份频率
- 定义执行快照保留周期
- 定义导出包保留周期
- 定义中间文件和缓存产物 TTL
- 定义审计日志保留周期

### 12.5 当前阶段部署要求

- 当前阶段后端服务以 Docker 容器方式部署
- 当前阶段文件优先保存在宿主机本地目录，并通过容器挂载卷访问
- 当前阶段设计必须保留未来迁移到云端对象存储和云端容器环境的兼容性

## 13. 模块验收标准

- 输入前提
- 操作步骤
- 预期结果
- 失败场景
- 审计要求
- 涉及统计结果时的数值容差或黄金样本对照

### 13.2 数据上传验收矩阵

| 场景 | 输入前提 | 预期结果 | 失败场景 | 审计要求 |

| 多源补充导入 | 已存在主数据版本 | 成功按主键映射补充字段 | 主键冲突、映射失败 | 记录合并模式和影响记录数 |

### 13.3 主键映射验收矩阵

| 场景 | 输入前提 | 预期结果 | 失败场景 | 审计要求 |
| --- | --- | --- | --- | --- |
| `微生态编号` 自动匹配 | 数据含唯一 `微生态编号` | 成功建立或匹配 `Subject` | 重复编号 | 记录自动映射结果 |
| `RD` 人工映射 | `微生态编号` 缺失 | 成功人工确认映射 | 映射冲突 | 记录映射人、原因、时间 |

### 13.4 分析任务验收矩阵

| 场景 | 输入前提 | 预期结果 | 失败场景 | 审计要求 |
| --- | --- | --- | --- | --- |
| 快速分析 | 已有发布版与品牌上下文 | 生成结果、主图、主表和简版解释 | 样本不足、变量不兼容 | 记录方法推荐与覆盖情况 |
| 高级分析 | 已有完整参数 | 生成完整统计结果和执行快照 | 方法前提不满足、任务超时 | 记录所有高级参数 |

### 13.5 派生变量发布验收矩阵

| 场景 | 输入前提 | 预期结果 | 失败场景 | 审计要求 |

### 13.6 高敏导出审批验收矩阵

- 黄金数据集
- 预期主统计量
- 数值容差
